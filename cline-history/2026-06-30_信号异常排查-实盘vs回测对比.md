# 2026-06-30 信号异常排查：实盘 vs 回测对比

> **问题**: 实盘引擎 23:51 产生金叉信号 (signal=1) 后，信号立刻消失，OM 从未执行开仓。
> **结论**: 信号算法正常，SM→OM 订单队列链路故障。

## 背景

实盘引擎于 2026-06-29 22:11 重新启动（Cline 修复 OM rawdata_show bug + LiveLogger 上线）。

运行期间 ETH 从 $1562 涨至 ~$1626，MA10 在 23:51 上穿 MA30，触发金叉 signal=1。

## 实盘观察

### 信号异常

从 `strategy_ticks.csv` 提取：

```

...后续全部 0，差价扩大到 +12.95（01:21），再无信号...
```

### 关键异常点

- 23:51 signal=1 后，MA10 一直在 MA30 上方（差价从 +0.67 → +12.95）
- 但 signal=1 只出现了一个 tick 就永远变回 0
- 即使 MA10 持续在 MA30 上方 1.5 小时，再无 signal=1
- OM 从未执行任何开仓（trades.csv 不存在）
- 无错误日志，引擎一直稳定运行

### 初步怀疑

OpenClaw 提出两种可能：
1. 信号算法要求持续交叉 N 个 tick 才确认（非瞬时交叉下单）
2. 竞态条件 bug（SM 线程 signal 状态被覆盖）

## 问题对比表

| 层面 | 回测 | 实盘 |
|:---|:---:|:---:|
| 信号算法 signalAlg | ✅ 25次交叉全部正确 | ✅ 23:51 正确产生 signal=1 |
| 信号传递 | ✅ 正常流入 orderAlg | ❌ SM→OM 队列接收端无响应 |
| OM 执行 | ✅ 25次交易全部执行 | ❌ 从未收到订单（0笔交易） |
| signal 持续 | ✅ 交叉后状态正确 | ❌ 1个tick后 signal 丢失 |
| trades.csv | ✅ 完整记录 | ❌ 文件不存在 |

## 结论

**信号算法没有 bug。** 回测在同一数据上运行正确，25 次交叉全部正常开平仓。

**问题在实盘的 SM→OM 订单队列链路。** 可能原因：
1. SM 线程的 `_run_algorithms()` 产生了订单但没推到 `self.oderqueue`
2. OM 线程的 `oderqueue.get()` 阻塞等待但队列无数据
3. SM 线程的 `processing_done` Event 同步机制导致数据处理提前结束
4. 回测和实盘使用了不同的 `config.ini`，orderAlg 参数可能有差异

**需要 Cline 排查：**
- `src/engine/strategy.py` 中 `_run_algorithms()` 后的 orderAlg 调用
- `src/engine/order_manager.py` 中 OM 线程的 run 循环入口
- `run_live.py` 中 SM/OM 线程的 start 顺序和 dp_queue 传递
